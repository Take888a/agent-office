# Agent Office 🏢

AI社員を雇って働かせるバーチャルオフィス。タスクを渡すと、Claude Agent SDK 製の
エージェントがドット絵の社員として出勤し、このプロジェクト上で実際に作業します。
頭上には「今なにしてるか」(💻コーディング中、🔍検索中…)がリアルタイム表示され、
完了すると報告書を提出してコーヒー休憩に入ります。

このリポジトリは「AI社員オフィス」の**テンプレート**です。clone して `claude` に
ログイン済みの環境で `npm run dev` すれば、そのまま自分のオフィスとして使えます。
社員名・オフィスレイアウト・服装などのカスタマイズポイントは `CLAUDE.md` を参照。

## 使い方

```bash
npm install
npm run dev
```

http://localhost:3000 を開き、上部フォームにタスクを入力して「雇う」を押すだけ。

- 社員はドアから出勤 → デスクで作業 → 完了したら ✅ を掲げてコーヒーマシンへ
- 右の社員名簿で進捗・報告書・参考コストを確認、「■ 中断」で作業を止められる
- オフィス右下のリソースモニタに AI 使用量を常時表示
  ([ccusage](https://github.com/ryoppippi/ccusage) 連携):
  5時間レート制限ウィンドウの消費(直近1週間のピーク比バー)、今日の使用量、
  **今月のコスト試算**(API単価換算。サブスク利用時は実請求ではなく参考値)
- `?demo` を付けると偽社員によるデモモード

## 認証

`claude` にログイン済みならそれだけで動きます(サブスク枠を消費、API課金なし)。
`ANTHROPIC_API_KEY` を設定すればAPI従量課金でも動作します。
サブスク認証での利用は自分専用ローカルツールの範囲に留めてください。

## 注意

- エージェントは `permissionMode: "bypassPermissions"` で動くため、
  確認なしにこのリポジトリのファイル変更・コマンド実行を行います。
  信用できるタスクだけを渡してください。
- 社員の状態はdevサーバーのメモリ上にあり、サーバー再起動で消えます。

## 構成

| ファイル | 役割 |
| --- | --- |
| `src/lib/office.ts` | 社員の雇用・進捗追跡・解雇 (Claude Agent SDK) |
| `src/app/api/employees/**` | 雇用/解雇 API |
| `src/app/api/agents/stream/route.ts` | SSE で社員状態を配信 |
| `src/components/OfficeCanvas.tsx` | Canvas によるオフィス描画・社員名簿 UI |
