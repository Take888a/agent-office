# Agent Office 🏢

常駐のAI社員たちが働くバーチャルオフィス。オーナー(あなた)が管理職に指示を出すと、
管理職が各チームの社員(Claude Code サブエージェント)に作業を割り振り、
ドット絵のオフィスで「誰が今なにをしているか」がリアルタイムに見えます。

このリポジトリは「AI社員オフィス」の**テンプレート**です。clone して `claude` に
ログイン済みの環境で `npm run dev` すれば、そのまま自分のオフィスとして使えます。

## 使い方

```bash
npm install
npm run dev
```

http://localhost:3000 を開くと、組織(office/org.json)どおりに社員が出勤してきます。

- **指示を出す**: 上部フォームからオーダーを送ると管理職が受け、適任の社員に
  Task ツールで委譲。各社員の頭上に作業内容(💻コーディング中 など)が出ます。
  プルダウンで特定の社員を直接指名することも可能。
- **オーダー履歴**: 右サイドバーで進捗・報告書・参考コストの確認、進行中の中断。
- **社員一覧** (`/employees`): 常駐社員の名簿。役割・得意分野・現在の状態。
- **組織編成** (`/org`): 人事担当に体制の要望を伝えると編成案を提案してくれる。
  内容を確認して「承認して適用」すると組織に反映(提案→承認→適用)。
  チームは役職別でも案件別でも編成できます。
- **リソースモニタ**(オフィス右下): [ccusage](https://github.com/ryoppippi/ccusage)
  連携で 5時間レート制限ウィンドウの消費・今日の使用量・**今月のコスト試算**
  (API単価換算。サブスク利用時は実請求ではなく参考値)を常時表示。

## 組織のしくみ

- **社員の実体は Claude Code のサブエージェント定義** `.claude/agents/<agent>.md`。
  このアプリの外(普段の claude セッション)からも同じ社員に委譲できます。
- **チーム編成は `office/org.json`**。手で編集しても、`/org` の人事経由でも変更可能。
- 管理職(オーケストレータ)はオーダーごとにセッションとして起動され、
  ロスター(チームと各社員の得意分野)を渡されて委譲判断をします。

## 認証

`claude` にログイン済みならそれだけで動きます(サブスク枠を消費、API課金なし)。
`ANTHROPIC_API_KEY` を設定すればAPI従量課金でも動作します。
サブスク認証での利用は自分専用ローカルツールの範囲に留めてください。

## 注意

- 社員は `permissionMode: "bypassPermissions"` で動くため、確認なしにこの
  リポジトリのファイル変更・コマンド実行を行います。信用できるオーダーだけを。
- オーダー履歴・社員の状態は dev サーバーのメモリ上にあり、再起動で消えます
  (組織定義はファイルなので消えません)。

## 構成

| パス | 役割 |
| --- | --- |
| `office/org.json` | チーム編成(管理職・人事・チームとメンバー) |
| `.claude/agents/*.md` | 社員の実体(サブエージェント定義) |
| `src/lib/org.ts` | 組織の読み書き・人事提案の適用 |
| `src/lib/office.ts` | オーダー実行・Task委譲の追跡・人事フロー |
| `src/lib/usage.ts` | AI使用量+コスト試算(ccusage 連携) |
| `src/app/api/**` | orders / org / システム統計 / SSE |
| `src/components/OfficeCanvas.tsx` | オフィス描画+オーダーUI |
| `src/components/EmployeeRoster.tsx` | 社員一覧ページ |
| `src/components/OrgEditor.tsx` | 組織編成ページ(人事オーダー+承認) |
